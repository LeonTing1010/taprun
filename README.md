# Claude Skills

Universal, project-agnostic skills for [Claude Code](https://claude.ai/claude-code). Each skill encodes a reusable development methodology — not project-specific commands.

## Skills

| Skill | Purpose | When to Use |
|-------|---------|-------------|
| [verify](verify/SKILL.md) | Multi-layer deterministic verification gates | After code changes, before PR |
| [constraint-driven-development](constraint-driven-development/SKILL.md) | RED→GREEN→Why workflow for business logic | Adding features, fixing business bugs |
| [writing-plans](writing-plans/SKILL.md) | Intent-verification task decomposition | Before complex implementations |
| [run-task](run-task/SKILL.md) | Semi-autonomous task execution from YAML | Delegating tasks to agents |

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/LeonTing1010/claude-skills/main/install.sh | bash
```

This clones the repo to `~/.claude-skills` and adds a `claude-skills` command to your shell.

## Usage

```bash
cd your-project
claude-skills init            # Link skills + generate CLAUDE.md template
```

Then edit `CLAUDE.md` to match your project. Done.

### Commands

```bash
claude-skills init              # Link skills + generate CLAUDE.md template
claude-skills init --skills     # Only link skills (don't touch CLAUDE.md)
claude-skills init --template   # Only generate CLAUDE.md template
claude-skills list              # List available skills
claude-skills update            # Pull latest version
claude-skills help              # Show help
```

## How It Works

```
┌─────────────────────────────┐
│  Skill (methodology)        │  ← Universal, lives in ~/.claude-skills/
│  "Run verification gates"   │
└──────────┬──────────────────┘
           │ reads
┌──────────▼──────────────────┐
│  CLAUDE.md (project config) │  ← Per-project, generated by `claude-skills init`
│  "pnpm typecheck"           │
│  "pnpm test"                │
└─────────────────────────────┘
```

Skills define **what to do** (the process). CLAUDE.md defines **how to do it** (the commands). Skills are symlinked, so `claude-skills update` updates all projects at once.

## Intent Router

Add this to your CLAUDE.md to enable automatic skill invocation:

```markdown
## Skill Router

| User Intent | Skill | Trigger |
|---|---|---|
| "check it" / after code changes | verify | Auto |
| "new feature" / "business bug" | constraint-driven-development | Auto |
| "write a plan" / complex task | writing-plans | Auto |
| task.yaml provided | run-task | Auto |
```

## Design Philosophy

Derived from first-principles engineering philosophy:

1. **Correctness must not depend on any single entity** → Cross-verification (verify)
2. **Correctness has a shelf life** → Temporal audit (constraint-driven-development)
3. **Make violations difficult, don't rely on memory** → Machine-enforced gates (verify)
4. **Reduce artifacts that need syncing** → Test = spec = documentation (constraint-driven-development)
5. **Split by intent, not by layer** → Intent-verification pairs (writing-plans)
