# walled-hosts.txt — hosts where server-side / cloud WebFetch STRUCTURALLY fails.
#
# These sites gate content behind an auth or bot/environment-verification wall,
# so a fetch from Claude Code's cloud proxy (WebFetch) can never see the
# logged-in page — it hits the wall instead. tap runs in YOUR own authenticated
# browser (credentials never leave the machine) and is the only tool that can.
#
# The webfetch-tap-router PreToolUse hook DENIES WebFetch to any host below and
# points Claude at tap instead. Match is exact host OR any subdomain
# (e.g. `weixin.qq.com` also blocks `mp.weixin.qq.com`).
#
# CRITERION for adding a host: cloud/server-side fetch essentially ALWAYS fails
# here (login/bot wall). Do NOT add hosts that serve useful public content to a
# plain fetch (e.g. github.com, most blogs, docs) — blocking those is a
# false-positive that just makes WebFetch worse. Precision is the whole point:
# an over-broad list gets the hook disabled.
#
# One registrable domain per line. Blank lines and `#` comments ignored.
