# 生意经纬图 · LOOM

> 任何生意拆到底就两个问句,没有第三个:**能拿到第一块钱吗**(需求)× **第二块钱比第一块省吗**(复利/护城河)。「好不好赚 × 守不守得住」就是这两问的连续刻度版——**好赚是经、守得住是纬**,单拉一根都是松线,两根**交织**才织出一块可卖可守会复利的布(1+1>2)。这把尺不告诉你"有没有护城河",它告诉你**护城河在不在你够得到的地方、用什么姿势去织**。

这个 skill 防的错误,是我反复犯过的那种:**把护城河架高一个抽象层去讲**("无 API"、"凭证局部性"),然后被一个理性对手随时间化解。LOOM 强制你把机会钉成**带坐标、带时间箭头、标了主语**的一个交织点(=一块布的命),而不是一句口号。

## 两轴 = 两把独立的尺

**X 轴「好不好赚」= 需求流动性(刻度 L0–L3)** — 不是"有没有人买"的是非题,是程度 = 速率 × 深度 × 摩擦。
用 X-尺量:深度(总买家/天花板)· 速率(尖峰 vs 站立)· 付费意愿(交易型"要做X" > 好奇)· 摩擦(够得着/在找/能否低成本闭环)· 可寻址(有入口/记得住名字)。每闸产出**刻度分量**,不是通过/否。
- **L0 无流**(没人要 / 摩擦∞够不着)· **L1 涓流-站立**(真需求但低速常青、被墙挡)· **L2 急流-尖峰**(事件驱动、窗口短)· **L3 深流**(大+持续+低摩擦+强意愿,**几乎必已被聚合商占**)。
- 边界判定题:L1/L2 = 有没有关窗倒计时?　L2/L3 = 持续且低摩擦?

**Y 轴「守不守得住」= 复利性 / 越做越省(刻度 C0–C3)** — 每多做一单,下一单成本/被抄难度怎么变(= MC 斜率)。
用 Y-尺量:复发(还会再来吗)· 沉淀物(留下可复用的东西吗 = 复利引擎)· 绕聚合商(关系能不走平台吗)· **抗克隆(B5,只是其中一格)** · 时间箭头(对手理性行动会不会关上这堵墙)。
- **C0 苦力/负复利**(每单从零,甚至越做越贵)· **C1 弱复利**(攒点模板,无锁定可被抄)· **C2 强复利**(沉淀物+复发+切换成本)· **C3 网络效应**(每用户让产品对下个用户更值钱)。
- 边界判定题:C0/C1 = 下一单省不省?　C1/C2 = 拿钱的克隆能不能一年内抹平你?

## 三条结构律(先内化,否则打分会骗你)

1. **两轴负相关,不是正交。** 让需求"好赚"的东西同时让它显眼值得占 → 涌入者毁掉"守得住"。⇒ 可达区是**反对角带**;**右上 L3×C3 是结构性禁区**(对新进入者永远空——若有开放的强护城河位,早被占了 = 聚合商墙律)。推论:**低流动(L1)是站立墙楔子的特性,不是缺陷**——没人抢才让你安静复利。
2. **点会漂,必须标时间箭头。** 靠"服务方不作为"(无 API / 门没关 / 聚合商没来建)的护城河一律是**倒计时**。尖峰需求 X 轴左漂(L2→L0),站立需求不漂。对每条声称的护城河,点名侵蚀它的理性行动者(服务方 API 化 / 克隆抄代码 / 聚合商吸信任 / 平台改 UI)+ 时间。
3. **复利要标主语:产品 vs 运营者。** 同一机会 Y 有两个值。追产品的 Y 常常必输(尤其 local-first/开放架构会把产品封顶在 C1);耐久在**运营者层**(done-for-you 剧本 + 客户关系 + 领域判断 = C2–C3)。

## 怎么跑

拿到用户的一句话点子(没有就先问)。按序做,每步说出证据,不靠语感:

1. **钉 X**:跑 X-尺,给出 L 刻度 + 理由(尤其判尖峰/站立,和是否已被聚合商占 → 是否撞禁区)。
2. **钉 Y(标主语)**:分别给"产品的 Y"和"运营者的 Y"两个 C 刻度。若架构(开放/local-first/无锁定)把产品封顶,明说封顶在哪一格。
3. **画时间箭头**:点名每条护城河的理性侵蚀者 + 两年后这点漂到哪。
4. **落区位 + 姿势**:
   - **L1–2 × C2(反对角甜点)= 真生意/楔子** → 唯一值得重仓。L2×C2 骑尖峰趁窗口转品牌;L1×C2 站立墙楔子。
   - **L2–3 × C0–1(右下)= 急流苦力/抽奖** → 拼速度抢一笔就跑,营收×30 该出手就出手,别投入感情。
   - **L0–1 × C2–3(左上)= 该造的品牌** → 复利好但没现成客户,必须自己造需求(换内容/社区引擎),**先找一只"有人买"的楔子启动**。
   - **L0 × C0(左下)= 别碰** → kill。
5. **交下一步**:如果落在值得做的区,把点子交给 `probe-first`(设计本周能发出的最省 costly probe);如果 X 轴证据薄弱,先交给 `demand-archaeologist`(按 6 轴歧视级联采真需求信号)再回来钉 X。

## 输出格式

```
点子:<一句话>
X 好不好赚:L?  ——<证据:深度/速率/摩擦/是否撞聚合商禁区>
Y 守不守得住:产品 C?  |  运营者 C?  ——<证据:复发/沉淀物/抗克隆>
时间箭头:<理性侵蚀者 + 两年后漂到哪>
区位 → 姿势:<四区之一 + 该怎么做>
下一步:<probe-first / demand-archaeologist / 重仓 / kill>
```

## 边界

- 这把尺**测机会的定位,不替代证据采集**(那是 demand-archaeologist)也不替代验证动作(那是 probe-first)。它回答"钉在哪、什么姿势",不回答"到底有没有人要"(去挖)或"这周怎么验"(去 probe)。
- 对 local-first / 开放架构的产品,正确输出不是"它有没有护城河"(结构上不会有),而是"**能不能让运营者的复利性攒得比工具被商品化更快**"。摩托车的护城河在骑手,不在车。
