# Tap — 本地优先的登录态浏览器自动化

## 它解决什么

你反复要在「已登录的网站」上做的杂活——每月拉发票 / 账单、登录后抓数据、定期巡检后台——不用每次都让 AI 实时驱动浏览器、反复烧 token。Tap 在你**自己真实的 Chrome** 里把任务**录制一次**，之后每次以**零 token 的确定性重放**执行，凭据从不离开你的机器。

## 四个动词（经 MCP 暴露给 Agent）

- `capture` —— 录制一次：AI 只在这一步参与，把页面编译成一个可重放的 `.plan.json` 程序。
- `run` —— 重放：确定性执行，**零 LLM token**。
- `verify` —— 健康检查：只读校验任务是否因网站改版而失效（漂移检测）。
- `mark` —— 确认：人工确认录制结果后提交。

已保存的任务作为 MCP 资源暴露为 `tap://{site}/{name}`，Agent 可直接调用。

## 为什么不同

- **零 token 重放**：传统浏览器 Agent 每次执行都重跑模型、烧 token；Tap 编译一次，之后纯数据分发。
- **本地优先**：运行在你的真实 Chrome，复用你已有的登录会话；cookies / 凭据**从不跨越信任边界**。
- **确定性 & 可验证**：每次重放结果一致；`verify` 在做之前先确认任务没坏。

## 如何连接（一次性）

WorkBuddy 用户级配置 `~/.workbuddy/mcp.json`：

```json
{
  "mcpServers": {
    "tap": {
      "command": "npx",
      "args": ["-y", "@taprun/cli", "mcp", "stdio"]
    }
  }
}
```

或在 WorkBuddy 界面：侧边栏 → 插件 → MCP 服务器 → 配置 MCP，可视化粘贴。连接后用自然语言描述需求即可。

> 首次连接 `npx` 会拉取约 50MB 引擎，首次握手可能需最多一分钟（可设 `MCP_TIMEOUT=120000` 放宽）。

## 典型场景

- 每月从银行 / 供应商后台拉发票、账单 PDF。
- 登录后抓取仪表盘数据、生成周期性报表。
- 定期检查某登录页状态、发帖 / 评论。

## 限制（如实说明）

- 运行环境：macOS / Windows + Chromium 系浏览器（Chrome / Edge / Brave / Arc）。
- 已登录站点需额外装 Tap 的 Chrome 扩展并点一次「Add to Chrome」（那一次点击是复用你登录会话的信任步骤，无法自动化）。
- 引擎闭源、以 npm 包 `@taprun/cli` 分发；CLI 与扩展 MIT 开源。

## 安全

- 本地优先：无云账号、无遥测、无授权心跳；程序以零权限沙箱运行。
- 凭据从不离开你的机器；Tap 不索取或传输你的密码。
